# EquipTrack

EquipTrack — українськомовна система обліку обладнання освітнього закладу. Вона веде приміщення, обладнання, відповідальних осіб, переміщення, ремонт, інвентаризації, списання, документи та журнал дій.

## Стек

- Next.js 15, React 19, TypeScript, Tailwind CSS;
- Surreal Cloud і SurrealDB JavaScript SDK;
- Zod для серверної валідації;
- bcryptjs і захищені cookie-сесії;
- sharp для стиснення фото в WebP.

Застосунок розгортається на Vercel. База даних працює окремо в Surreal Cloud. Фото й документи не надсилаються до зовнішніх файлових сервісів: двійкові дані зберігаються у записах таблиці `file` в SurrealDB.

## Налаштування Surreal Cloud

1. Створіть інстанс і базу в Surreal Cloud.
2. Скопіюйте `.env.example` у `.env.local`.
3. Заповніть значення:

```dotenv
SURREAL_URL=wss://your-instance.surreal.cloud
SURREAL_NAMESPACE=equiptrack
SURREAL_DATABASE=production
SURREAL_USERNAME=your_username
SURREAL_PASSWORD=your_password
AUTH_SECRET=long-random-secret
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

У Vercel створіть ті самі змінні середовища для Production і Preview. Не додавайте `.env.local` до Git.

## Команди

```bash
npm install
npm run db:migrate
npm run storage:setup
npm run db:seed
npm run setup
npm run dev
npm run build
```

`npm run setup` застосовує схему та створює дані, якщо їх ще немає. За відсутності змінних середовища застосунок показує сторінку `/setup` без фіктивних робочих даних.

## Початкові облікові записи

- `admin@equiptrack.local` / `admin123` — адміністратор;
- `manager-1@equiptrack.local` / `manager123` — менеджер обліку;
- `staff1@equiptrack.local` / `staff123` — працівник.

Змініть ці паролі в реальному середовищі одразу після першого запуску.

## Ролі

- `staff` переглядає приміщення й обладнання, подає заявки та повідомляє про несправності;
- `inventory_manager` веде обладнання, приміщення, переміщення, ремонти, аудити й пропозиції списання;
- `admin` керує користувачами, довідниками, списанням і журналом дій.

Сервер не дозволяє деактивувати останнього активного адміністратора.

## Дані та workflow

Основні записи SurrealDB: `user`, `session`, `building`, `room_type`, `room`, `category`, `supplier`, `equipment`, `file`, `movement`, `transfer_request`, `repair`, `audit`, `audit_item`, `writeoff_request`, `notification`, `audit_log`.

Створення обладнання одразу створює його перший запис руху. Переміщення, ремонт, аудит і списання зберігають історію; списане обладнання не можна використовувати повторно. Каталоги, журнали та документи отримують дані з сервера із пагінацією.

## Файли в БД

Підтримуються JPEG, PNG, WebP і PDF до 3 МБ. Фото стискаються до WebP, для списків створюється окрема мініатюра. Дані фото й документів не входять у спискові запити: їх повертає лише захищений маршрут `/api/files/[fileId]/[variant]` після перевірки сесії та права доступу.

## Інтерфейс

Усі сторінки мають український інтерфейс. У UI немає SVG-іконок або SVG-ілюстрацій: візуальна мова побудована на CSS, текстових позначках і растрових фото з SurrealDB. Навігація — правий текстовий індекс інвентарної картотеки, який на мобільних екранах відкривається нижньою панеллю.

## Перевірка

Вимоги проєкту виключають пакет автоматизованих тестів. Перед розгортанням запускайте:

```bash
npm run check
```

Після підключення Surreal Cloud вручну перевірте вхід трьома ролями, створення обладнання з файлом, передачу, повідомлення про ремонт, аудит, списання, доступ до документів і режим без змінних середовища.
